Remote access — certificate management

Generate and manage the remote-access secret material (CA, device .p12 bundles, the login passphrase) without dropping to the tclaude remote-access CLI.

Material
Cert valid for
The host names / IPs a device may dial without a certificate-name mismatch (the server cert's SANs).
Adds SANs to the server cert (keeps the existing CA), so a public URL / new name verifies — installed devices keep working, no reinstall. Restart agentd to serve the updated cert.
Devices
Issues a new .p12 from the existing CA, then download it to install on the device.
First-time setup / regenerate material

Generate the CA + server cert + first device, set the login passphrase, and (optionally) enable the listener. If material already exists, regenerate rotates everything including the CA — this invalidates every installed device and they must reinstall.
